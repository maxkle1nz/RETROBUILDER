# Roadmap & Pending Work

Current baseline: version `0.6.1`, Node `>=22.0.0`, active OMX runtime in `src/server/omx-runtime.ts`, SPECULAR showcase browser verification in place, THE BRIDGE defaulting to `gpt-5.5`, local API and Git safety controls in place, and `npm run verify:readiness` as the canonical local readiness gate.

Verification for the current release baseline can be regenerated through `npm run verify:specular`, `npm run verify:readiness`, and the optional `npm run verify:live-kompletus-browser` lane. Local reports and screenshots remain machine artifacts and are not committed to the docs.

This roadmap focuses on work that remains open in the current codebase.

---

## 1. Full Visual SPECULAR Verification
Goal: verify in a real browser that backend state is represented correctly from session creation through KOMPLETUS and OMX completion.

Status: complete for deterministic browser verification, with focused live external-provider handoff coverage

Delivered:
- SPECULAR showcase renders a browser-readable verification manifest (`#rb-specular-truth`) with session id, surface count, per-node design profile, selected 21st references, selected variant, gate status, and score.
- `tests/ui-specular-showcase-chromium-cdp.ts` opens `/specular/showcase/:sessionId` in real Chromium/CDP and verifies:
  - DOM surface ids match backend manifest node ids
  - every surface is anchored to `designProfile: "21st"`
  - selected 21st references are present
  - old dark/glass/card token vocabulary is absent
  - product hero/actions render
  - 390px mobile viewport does not horizontally overflow
- `npm run verify:specular` now self-starts Retrobuilder when needed and includes this SPECULAR browser smoke when local Chromium is available.
- The same full verify command now runs a Node/TS Chromium verification matrix without Python `websockets`, covering shell render, design-gate block -> UIX correction, valid UIX -> builder mode, resume hint, and resume-chat continuation.
- The Chromium verification matrix also includes a deterministic mechanic CRM full journey: backend session creation, SPECULAR generate/save, browser showcase manifest, OMX handoff, reload, and BU1LDER reentry preservation.
- When `RETROBUILDER_BROWSER_ARTIFACT_DIR` is set, the Chromium matrix saves PNG artifacts for full-journey showcase, builder handoff, builder reentry, and SPECULAR showcase desktop/mobile views.
- `npm run verify:live-kompletus-browser` provides an optional live external-provider browser harness. It is gated by `RETROBUILDER_RUN_LIVE_E2E=1`, runs KOMPLETUS SSE, persists the generated session, opens SPECULAR in Chromium, captures PNG artifacts, and hands the session into BU1LDER.
- A credentialed local run on `2026-04-26` generated a mechanic CRM/site graph, grounded 12 modules, passed SPECULAR CREATE at `100%` across 4 surfaces, captured `live-kompletus-specular-showcase.png`, captured `live-kompletus-builder-handoff.png`, and verified BU1LDER handoff.
- Evidence from that focused run is regenerated by the commands above and, when artifact retention is enabled, written to ignored local artifact and report directories.

Open work:
- optional CI or manual artifact retention for the live external-LLM lane on a credentialed runner
- broader backend-vs-UI verification matrix for every critical event beyond the focused SPECULAR/BU1LDER handoff

Why this matters:
- current tests already validate transport and contract behavior
- SPECULAR showcase output is now verified in a real browser
- the remaining work is deciding where credentialed live verification should run and how artifacts should be retained

## 7. Local Security / Repo Reputation
Goal: keep local AI/config/build capabilities explicit and keep generated runtime artifacts out of the public Git history.

Status: complete for the current security and repository-hygiene baseline

Delivered:
- `/api/ai`, `/api/config`, `/api/m1nd`, `/api/omx`, and codebase import share `requireLocalApiToken` when `RETROBUILDER_LOCAL_API_TOKEN` is configured, and non-loopback binds require the token at startup.
- Browser API clients send the central `X-Retrobuilder-Token` header to AI/config/OMX/m1nd/import routes.
- The browser token helper no longer reads persistent `localStorage`; manually supplied tokens are in-memory only.
- THE BRIDGE local Codex JSON fallback is disabled by default and requires `RETROBUILDER_ENABLE_LOCAL_CODEX_FALLBACK=1`.
- The local Codex fallback no longer passes the dangerous sandbox-bypass flag and writes a read-only sandbox config.
- Server source no longer carries personal `/Users/...` fallback paths for Bridge, OpenClaw profiles, or the 21st catalog.
- `postcss` was lifted to a non-vulnerable lockfile version after `npm audit` reported `GHSA-qx2v-qp2m-jg93`.

Open work:
- if Retrobuilder is intentionally exposed beyond loopback, document the operational token rotation workflow
- split the current large dirty worktree into logical commits before opening a public PR
