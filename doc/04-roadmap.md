# Roadmap & Pending Work

Current baseline: working tree @ version `0.6.1`, Node `>=22.0.0`, active OMX runtime in `src/server/omx-runtime.ts`, SPECULAR showcase browser-truth proof in place, THE BRIDGE defaulting to `gpt-5.5`, local API/Git security guardrails in place, and `npm run verify:readiness` as the canonical local readiness gate.

Fresh evidence for the latest release-readiness slice is regenerated through `npm run verify:specular`, `npm run verify:readiness`, and the opt-in `npm run verify:live-kompletus-browser` lane. Raw reports/screenshots remain local artifacts, not committed docs.

This roadmap only lists the work that is still genuinely open from the current codebase.

---

## 1. Full Visual Specular Truth Test
Objective: prove, in a real browser, that backend truth is translated correctly from session creation through KOMPLETUS and OMX completion.

Status: complete for deterministic truth and focused live external-provider handoff truth

Completed now:
- SPECULAR showcase renders a browser-readable truth manifest (`#rb-specular-truth`) with session id, surface count, per-node design profile, selected 21st references, selected variant, gate status, and score.
- `tests/ui-specular-showcase-chromium-cdp.ts` opens `/specular/showcase/:sessionId` in real Chromium/CDP and verifies:
  - DOM surface ids match backend truth manifest node ids
  - every surface is anchored to `designProfile: "21st"`
  - selected 21st references are present
  - old dark/glass/card token vocabulary is absent
  - product hero/actions render
  - 390px mobile viewport does not horizontally overflow
- `npm run verify:specular` now self-starts Retrobuilder when needed and includes this SPECULAR browser truth smoke when local Chromium is available.
- The same full verify command now runs a Node/TS Chromium workbench matrix without Python `websockets`, covering shell render, design-gate block -> UIX correction, valid UIX -> builder mode, resume hint, and resume-chat continuation.
- The workbench matrix now also includes a deterministic mechanic CRM full journey: backend session creation, SPECULAR generate/save, browser showcase truth manifest, OMX handoff, reload, and BU1LDER reentry preservation.
- When `RETROBUILDER_BROWSER_ARTIFACT_DIR` is set, the Chromium matrix saves PNG artifacts for full-journey showcase, builder handoff, builder reentry, and SPECULAR showcase desktop/mobile views.
- `npm run verify:live-kompletus-browser` now provides an opt-in live external-provider browser harness. It is gated by `RETROBUILDER_RUN_LIVE_E2E=1`, runs KOMPLETUS SSE, persists the generated session, opens SPECULAR in Chromium, captures PNG artifacts, and hands the session into BU1LDER.
- A credentialed focused live run on `2026-04-26` generated a mechanic CRM/site graph, grounded 12 modules, passed SPECULAR CREATE at `100%` across 4 surfaces, captured `live-kompletus-specular-showcase.png`, captured `live-kompletus-builder-handoff.png`, and verified BU1LDER handoff.
- Focused run evidence is regenerated by the commands above and, when artifact retention is enabled, written under ignored local artifact/report directories.

Still needed:
- optional CI/manual artifact retention for the live external-LLM lane on a credentialed runner
- broader backend-vs-UI truth matrix for every critical event beyond the focused SPECULAR/BU1LDER handoff

Why this matters:
- current tests validate transport/contracts well
- SPECULAR showcase truth is now browser-proved
- the remaining gap is no longer live proof; it is productizing where that expensive credentialed proof runs and how artifacts are retained

## 7. Local Security / Repo Reputation
Objective: make sensitive local capabilities explicit and keep generated/runtime noise out of the public history.

Status: complete for the current guardrail slice

Completed now:
- `/api/ai`, `/api/config`, `/api/m1nd`, `/api/omx`, and codebase import share `requireLocalApiToken` when `RETROBUILDER_LOCAL_API_TOKEN` is configured, and non-loopback binds require the token at startup.
- Browser API clients send the central `X-Retrobuilder-Token` header to AI/config/OMX/m1nd/import routes.
- The browser token helper no longer reads persistent `localStorage`; manually supplied tokens are in-memory only.
- THE BRIDGE local Codex JSON fallback is disabled by default and requires `RETROBUILDER_ENABLE_LOCAL_CODEX_FALLBACK=1`.
- The local Codex fallback no longer passes the dangerous sandbox-bypass flag and writes a read-only sandbox config.
- Server source no longer carries personal `/Users/...` fallback paths for Bridge, OpenClaw profiles, or the 21st catalog.
- `postcss` was lifted to a non-vulnerable lockfile version after `npm audit` reported `GHSA-qx2v-qp2m-jg93`.

Still worth improving:
- if Retrobuilder is intentionally exposed beyond loopback, document the operational token rotation workflow
- split the current large dirty worktree into logical commits before opening a public PR
